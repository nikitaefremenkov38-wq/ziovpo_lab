# UML и ER-диаграммы

## UML

UML (Unified Modeling Language) используется для визуального описания структуры и поведения программной системы.

Основные типы UML-диаграмм:

- Диаграмма классов показывает классы, поля, методы и связи между объектами.
- Диаграмма вариантов использования описывает роли пользователей и сценарии взаимодействия с системой.
- Диаграмма последовательности показывает обмен сообщениями между объектами во времени.
- Диаграмма активности описывает поток действий и условия переходов.
- Диаграмма состояний показывает жизненный цикл объекта и переходы между состояниями.
- Диаграмма компонентов описывает программные модули и зависимости между ними.
- Диаграмма развертывания показывает, на каких узлах размещаются компоненты системы.

Для серверной части проекта особенно полезны диаграммы классов, последовательности, компонентов и развертывания: они помогают описать доменную модель, REST-сценарии, слои приложения и инфраструктуру.

## ER-диаграммы

ER-диаграмма (Entity-Relationship) описывает структуру реляционной базы данных через сущности, атрибуты и связи.

Основные элементы:

- Сущность соответствует таблице базы данных.
- Атрибут соответствует колонке таблицы.
- Первичный ключ уникально определяет запись.
- Внешний ключ связывает запись с другой таблицей.
- Связи показывают кардинальность: один-к-одному, один-ко-многим, многие-ко-многим.

В проекте ER-модель описывает аттракционы, расписания, посетителей, билеты, техническое обслуживание и пользователей системы. Такая диаграмма нужна, чтобы заранее увидеть ограничения целостности, уникальные поля и связи между таблицами.
